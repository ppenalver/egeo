Design and behaviour

There are multiple button designs that are related with different scenarios. We use CTA buttons with a blue solid background to guide the user along the product, remarking with them the principal action of each screen. Only must be one CTA per screen.

Secondary buttons are used for non-principle actions and could live together with other secondary buttons although is not recommended if possible.

Third buttons are usually used for cancel actions on modals although could be used for other purposes.

Critical buttons are used only to remark actions that, dua its criticism, could create serious problems (E.g. delete actions or critical configuration changes).

Each of these four types works with main actions on a screen but on different levels of importance. Ideally, only one of each at most will exist in the same place. Besides, all these types of buttons could be accompanied by an icon after or before the text.

A five type of button exists. This button, unlike the others, is designed to be used multiple times on the same screen (E.g. in a toolbar). These are only-icon buttons.

Solid buttons look heavy on purpose. They direct the user’s attention to the primary action the application is suggesting that the user take.

Outline buttons (and lightgray solid ones) provide a lighter weight button style. They are used to indicate a secondary action visually that compliments a primary action or to reduce visual noise when there are many action of equal importance on the page.

Usage and best practices

Below are the good practices that should guide the uses of the component to maintain coherence in the design system and the spirit with which they were sustained.

  • Never use more than one CTA button at the same time. Each screen must have one only CTA button available. This is to guide the user about the purpose of the screen with no doubts.
  • Critical buttons should be only used when the result of the action could create unexpected or critical issues. Are not related to errors.
