Links are button-like items that allow to place them on situations where buttons are too big, too remarkable, or need to appear so many times that is a bad design decision to use them. Links are subtle, user understands them quite well, and are more flexible.

Design and behaviour

Links are exactly that, links, and must have the link behavior. So, links use the CTA colors (usually blue and red) and show a hand when the user hovers over it (CSS cursor: pointer).

Links can be accompany an icon to emphasize the action or to reduce the time the user needs to understand the action. E.g. the plus icon to the action Add.

Links should have a security hover space around them to facilitates the user to click on it.

Usage and best practices

Below are the good practices that should guide the uses of the component to maintain coherence and consistency in the design system and the spirit with which they were sustained.

  • Use Links when the action is clearly secondary or even less importante.
  • Try to be as clear and concise as possible. Links must have two words or less and must describe the action as well as possible. E.g. Add item, Delete, More info...
  • Usually, the link texts are begun by a verb that identifies the action. E.g. Add, Delete, Edit. However, exceptionally, can use other words. E.g. More.
